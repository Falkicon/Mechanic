Run a full code review before merging or releasing.

**Skills**: [s-lint](../skills/s-lint/SKILL.md), [s-audit](../skills/s-audit/SKILL.md), [s-test](../skills/s-test/SKILL.md), [s-debug](../skills/s-debug/SKILL.md), [s-clean](../skills/s-clean/SKILL.md)

1. **Lint**: Run code quality checks (see s-lint for details).
2. **Audit**: Run security, complexity, and deprecation analysis (see s-audit for details).
3. **Test**: Verify all tests pass (see s-test for details).
4. **Debug**: Check for runtime errors via reload loop (see s-debug for details).
5. **Clean**: Check for dead code and stale docs (see s-clean for details).
6. **Report**: Summarize findings and readiness for merge/release.
